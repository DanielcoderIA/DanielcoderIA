### `FREEFLOW CRM` &nbsp;—&nbsp; SaaS Freemium "Lead-to-Cash"

**Plataforma de gestión financiera integral para agencias y freelancers.**
Automatiza el ciclo comercial completo — desde la captación del lead hasta el cobro final.

```
FLUJO  →  Cliente  ▸  Propuesta  ▸  Proyecto  ▸  Factura  ▸  Cobro
```

**LOGRO CLAVE:** Ciclo de vida del cliente 100% centralizado. Facturación automatizada vía Stripe con monitoreo de KPIs en tiempo real, eliminando la dependencia de herramientas externas costosas.

| MÓDULO | IMPLEMENTACIÓN TÉCNICA |
|---|---|
| 💳 **Monetización real** | Stripe Checkout + Billing Portal · Plan Free / Pro $19/mes · pagos activos en producción |
| 🔄 **Suscripciones recurrentes** | Webhooks `checkout.session.completed` · `customer.subscription.deleted` · manejo completo de estados |
| 🔐 **Autenticación multitenant SSR** | Row Level Security en Supabase — aislamiento total de datos por tenant, sin exposición cruzada |
| ⚡ **Performance arquitectural** | Server Components + Streaming — hidratación mínima del cliente, cero layout shift |
| 📊 **KPIs en tiempo real** | Dashboard live vía Supabase Realtime — actualizaciones push, sin polling ni cron jobs |

```
STACK  →  Next.js 15 · React 19 · TypeScript 5 · Supabase · PostgreSQL · Stripe · Tailwind CSS 4 · Radix UI
```

### `DOCUMENTE` &nbsp;—&nbsp; Motor RAG de Alta Precisión

<img src="https://img.shields.io/badge/●_STATUS-LIVE-CC0000?style=flat-square&labelColor=000000"/>
<img src="https://img.shields.io/badge/MERCADO-Legal_%26_Contable_LATAM-333333?style=flat-square&labelColor=111111"/>
<img src="https://img.shields.io/badge/EMBEDDINGS-Transformers.js_384D-555555?style=flat-square&labelColor=111111"/>
<img src="https://img.shields.io/badge/NEXT.JS-15-ffffff?style=flat-square&logo=nextdotjs&logoColor=000000&labelColor=ffffff"/>

**Motor de análisis documental con IA especializado en el mercado legal y contable colombiano.**
Procesa PDFs complejos con búsqueda vectorial semántica — consultas en lenguaje natural sobre múltiples documentos.

```
FLUJO  →  PDF  ▸  Chunking  ▸  Transformers.js  ▸  pgvector  ▸  Síntesis LLM
```

> **Arquitectura de costos:** Los embeddings 384D se generan mediante **Transformers.js en el runtime de Node.js** — sin APIs externas en el pipeline de datos. pgvector ejecuta la búsqueda semántica nativamente en PostgreSQL. Gemini se usa **únicamente para razonamiento final**, reduciendo el consumo de tokens hasta un 80% frente a un pipeline 100% externo.

**LOGRO CLAVE:** Pipeline RAG con embeddings generados vía Transformers.js (Node.js runtime) y búsqueda nativa en PostgreSQL — cero dependencia de OpenAI/Ada. Recursive Character Splitting + Re-ranking semántico para textos legales colombianos con latencia de milisegundos.

| MÓDULO | IMPLEMENTACIÓN TÉCNICA |
|---|---|
| 🔍 **Búsqueda semántica local** | pgvector en PostgreSQL — similitud coseno en ms · sin latencia de red hacia APIs de terceros |
| 🤖 **Embeddings en el Edge (costo ≈ $0)** | Vectorización 384D vía **Transformers.js** en Node.js runtime — independencia total de OpenAI/Ada |
| 📄 **Chunking para documentos legales** | Recursive Character Splitting con overlap dinámico + Re-ranking calibrado para español jurídico colombiano |
| ⚖️ **Orquestación híbrida de LLMs** | Groq/Llama 3 (baja latencia) ↔ Gemini (razonamiento profundo, multi-doc) · enrutamiento por complejidad |
| 🛡️ **Seguridad de datos** | RLS en Supabase — vectores y documentos blindados por sesión · threshold de similitud ajustado |

```
STACK  →  Next.js 15 · React 19 · TypeScript · Supabase · pgvector · Clerk · Transformers.js · Groq · Gemini
```
